# ספר טלפונים סידב – אפליקציית אנדרואיד

אפליקציה מבוססת Capacitor שעוטפת דף HTML/JS שמתחבר לאותו מקור נתונים (MyJSON) כמו האתר.
כל עדכון שנעשה באתר (או באפליקציה עצמה) מופיע אוטומטית בשני המקומות, כי שניהם קוראים וכותבים לאותו רשומת API.

## מבנה הפרויקט
```
cidev-phonebook-app/
├── www/index.html       ← כל האפליקציה (HTML+CSS+JS), מותאמת למגע ולמסך טלפון
├── package.json         ← תלויות Capacitor
├── capacitor.config.json
├── codemagic.yaml        ← קונפיגורציית בילד מוכנה ל-Codemagic
└── README.md
```

## מה שונה מגרסת האתר
- עיצוב מותאם מגע: אקורדיון מחלקות שנפתח/נסגר, שבבי סינון גוללים, סרגל עליון קבוע.
- נוסף כפתור **התקשר** (חיוג ישיר) ליד כל איש קשר עם טלפון.
- הוסרו הגדרות ההדפסה (לא רלוונטיות באפליקציה).
- שאר הפיצ'רים זהים: חיפוש, סינון, מועדפים, וואטסאפ, מייל, הוספה לאנשי קשר בטלפון (vCard), הוספה/עריכה/מחיקה מוגנת בסיסמה (`cidev123` – כדאי לשקול להחליף לסיסמה חזקה יותר בהמשך, כרגע היא כתובה גלויה בקוד הצד-לקוח).

## הרצה מקומית לפני העלאה ל-Codemagic
צריך Node.js מותקן במחשב (לא בסביבה הזו – יש להריץ את הפקודות הבאות אצלך):

```bash
cd cidev-phonebook-app
npm install
npx cap add android      # יוצר את תיקיית android/ בפעם הראשונה
npx cap sync android
```

זה ייצור תיקיית `android/` מלאה. מומלץ להוסיף גם אותה ל-git (כלומר, לא ל-gitignore),
כדי ש-Codemagic יוכל לבנות ישירות בלי להריץ `cap add` בכל בילד (למרות שה-`codemagic.yaml` המצורף גם יודע להתמודד עם זה לבד אם התיקייה לא קיימת).

## חיבור ל-Codemagic
1. היכנס ל-https://codemagic.io/apps ולחץ על **Add application**.
2. חבר את הריפו מ-GitHub.
3. Codemagic יזהה את `codemagic.yaml` שבשורש הפרויקט ויציע את ה-workflow `android-phonebook`.
4. הרץ בילד (Start new build). בסיום יהיה קובץ APK להורדה תחת Artifacts.
5. ה-APK בבילד הזה הוא **debug** (לא חתום) – מספיק להתקנה ובדיקה ישירה על מכשיר. אם בהמשך תרצה להפיץ ב-Play Store, נצטרך להוסיף חתימת release (keystore) ולעדכן את הסקריפט ל-`assembleRelease`/`bundleRelease` – זה משהו שאפשר להוסיף כשתגיע לשלב הזה.

## הרשאות
Capacitor מוסיף אוטומטית הרשאת אינטרנט (INTERNET) לפרויקט האנדרואיד שנוצר, הנדרשת לקריאה/כתיבה מול MyJSON.

## אייקון האפליקציה
בתיקיית `assets/` נמצאים שלושה קבצים שהופקו מהלוגו שנתת (הוסר הרקע העגול הלבן והצל – נשאר רק הלוגו הכחול):
- `icon-only.png` – הלוגו על רקע לבן מלא (משמש לאייקון "רגיל"/legacy ולאייקון Google Play).
- `icon-foreground.png` – הלוגו בלבד, שקוף, ממורכז בתוך אזור בטוח (safe zone) – משמש כשכבת החזית של האייקון האדפטיבי של אנדרואיד.
- `icon-background.png` – ריבוע לבן אחיד – משמש כשכבת הרקע של האייקון האדפטיבי.

כדי לייצר מהם את כל גדלי האייקונים הנדרשים (mipmap-mdpi עד xxxhdpi, אייקון אדפטיבי, ואייקון ל-Play Store) יש שני מסלולים:

**אוטומטי (מומלץ) – דרך הסקריפט המצורף:**
```bash
npm install
npx cap add android      # אם עוד לא נוצרה תיקיית android
npm run icons             # מריץ את @capacitor/assets ומייצר את כל גדלי האייקונים לתוך android/
npx cap sync android
```
גם ה-`codemagic.yaml` המצורף כולל את השלב הזה אוטומטית בכל בילד, כך שגם בלי להריץ מקומית – Codemagic ידאג לזה.

אם בעתיד תרצה לשנות את צבע הרקע של האייקון האדפטיבי (למשל לכחול המותג במקום לבן), פשוט ערוך את `icon-background.png` או שנה את הפרמטר `--iconBackgroundColor` בסקריפט `icons` שב-`package.json`.

## התאמות עתידיות אפשריות
- הוספת אייקון ומסך פתיחה (splash) מותאמים אישית – דרך `npx cap` + עדכון קבצי המשאבים באנדרואיד.
- מעבר מ-localStorage (מועדפים, מצב כהה) ל-Capacitor Preferences API אם ירצה שהנתונים האלה יהיו יציבים יותר בין עדכוני אפליקציה.
- הרשאת אנשי קשר אמיתית (Capacitor Contacts plugin) במקום קובץ vCard, לשמירה ישירה לרשימת אנשי הקשר בלי מסך ביניים.
